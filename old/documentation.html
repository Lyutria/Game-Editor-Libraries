ltCANVAS API Documentation

Introduction

ltCanvas Tools was written by DarkParadox on the Game-Editor.com forums and is a collection of about 2500 lines of code for functions that are useful in drawing on canvases in various ways or quickly prototyping very usable GUI elements.

I've pretty much stopped adding videos, and I've added the new 6.0 functions.

If you're going to be editing anything, like adding your own functions, putting in more GUI color themes, or just looking through the code, you should probably grab an external code editor instead of GE's for it. There are far too many lines for GE's editor to be useful. I suggest something like the amazing Komodo Edit (for Windows, Mac, and Linux) or the ever-popular Notepad++ (Windows only). I personally use an editor called VIM, but I don't suggest you use it without being a total computer power-user.
General Functions
drawText Draw plain text on the topleft of a canvas.
Syntax: void drawText(char* text);
Usage: drawText("Text!");
Variables: char* text A string for the text to display on the canvas.
When writing the text, typing one of these (minus quotes) will do special actions:
"\n" - Will move the text onto a new line
"\a" - an "\a" follow by three numbers will change the following text to the corresponding color "\a[red][blue][green]", with "\a000" being black, and "\a999" being white.
"\t" followed by a number 0-9 will change the transperancy level with "\t0" being not transperant and "\t9" being fully transperant. You can't set transperancy any other way. (buggy)
"\v[-/+][x][-/+][y]", will shift text specific directions based on the input.

Example: drawText("The text after THIS \a009is blue and this is \v-1+2downleft");

These color items don't work with buttons or other GUI elements found later in the documentation as it can't correctly compute the text's width when they are used. I'll probably fix this eventually.

General HOWTO for GUI functions:
Each of these GUI function belongs to a group that are used to quickly make a type of GUI element, like a button or a text box.

The last parameter of each of these functions is usually a set of 12 different RGB values, but you aren't supposed to type them out like that. There are 4 special alias/defines that make it easy to get good control over the appearance of your GUI elements. They are:
  LUM(X) : Creates a grayscale style element
  RGB(X, Y, Z) : Creates an RGB colored element
  THEME(theme_name) : Creates a specially themed element
  THEME(CURTHEME): A special theme setting, defaulted to "theme_eos", a subtle green theme.
    About this one, you use a special function "setCURTHEME" to change the value.
    So, you can set all your GUI elements to be "THEME(CURTHEME)", and then, if you want to change
    the theme of ALL of your GUI elements, you can use this function:
        setCURTHEME(theme_name);
    Where "theme_name" is any theme you've added or any in there already.
    The CanvasEx demo on my Black Hole of Codes topic has this as an example.

They are used within one of these functions like so:
  label("Label Text", RGB(255,255,255));

Included are SIX themes to start, and they are:
theme_dark, theme_grey, theme_light, theme_sepia, theme_eos, and theme_blank
Like above, we can use them by doing:
  label("Label Text", THEME(theme_eos));

THESE ARE DEFINED IN THE FOLLOWING CODE DOCUMENTATION AS THE TEXT "[COLORTYPE]"
 
You can make your own themes and adjust the default theme for CURTHEME by looking the global code right after the #DEFINE section. Themes are an array of 12 integers for 4 RGB values. The sets are, in order:
  BORDER/MOUSE OVER HIGHLIGHT
  UNTOUCHED BUTTON COLOR
  BACKGROUND COLOR
  TEXT COLOR

EDITABLE TEXT BOX ELEMENT
Creates a text box, limitable in many forms.
To stop typing in a textBox, you have to press ENTER or click the arrow.
No variable creation needed, but will take over SEVERAL variables that you normally wouldn't think to use. Here are the variables which you probably shouldn't access freely:
textbox.text (stores the string from the textbox)
textbox.animpos (Whether or not the textbox is being typed in. 1 is true)
textbox.animindex (Horizontal position of the typing cursor)
view.animpos (Stores a value to manage the cursor's blinking speed)
Normally you wouldn't use these anyways, so this shouldn't be a problem!
textBox DRAW ACTOR EVENT
Syntax: void textBox([COLORTYPE])
Usage: GUI;
textBox(LUM(50));
Variables: Actor* actor Place the name of the current actor prefixed with a "&" here.

textBoxMouseDown MOUSE DOWN EVENT
Syntax: void textBoxMouseDown()
Usage: GUI;
textBoxMouseDown();
Variables: Actor* actor Place the name of the current actor prefixed with a "&" here.

textBoxKeyDown KEY DOWN EVENT (SET TO "ANY KEY", NO REPEAT)
Syntax: void textBoxKeyDown(int limit, char* modes)
Usage: textBoxKeyDown(0, "a");
Variables: Actor* actor
int limit		 Current actor prefixed by "&"
Limit on how many letters entered, 0 is infinite.
char* modes A list of several letters that allow different types of characters in the text box. They can be combined in any way you want, like "lws"

"a" - ANY character
"l" - All letters ("ABCabc")
"n" - All numbers ("123")
"s" - All symbols (".,@#")
"w" - Whitespace (Like a single space " ")

So, for example, "lns" would allow all characters except for spaces.
